Audio Data Source Options for a Better LinguaGuessr

The current Lingua Libre library is easy to ship, but many clips are isolated words. For a game about listening, 5-10 second phrases or short sentences would feel more human, give players enough phonetic texture to guess, and make reveal screens more interesting.

Recommendation: use Mozilla Common Voice as the first production replacement for the main language game, keep the Speech Accent Archive only for Accent Mode, and evaluate FLEURS as a controlled parallel-sentence mode. If we want endangered-language flavor, treat archives like Pangloss/ELAR as curated editorial additions, not the bulk game corpus.

Recommended Product Shape

Main Game: “Real Sentences”

Replace Lingua Libre single-word clips with Common Voice validated sentence clips. Target 5-10 seconds, avoid clips with obvious place names, and show the transcript only as the existing score-lowering hint.

Daily Challenge

Pull from a prebuilt same-origin manifest so every daily seed is deterministic and fast. Include duration, transcript, source, language metadata, speaker metadata when allowed, and attribution if required.

Accent Mode

Keep the Speech Accent Archive concept, but label it explicitly as “English accent mode.” It should never be mixed with native-language guessing because the audio language is always English.

Parallel Sentence Mode

Use FLEURS for a mode where different languages read semantically parallel prompts. This makes comparison fairer and teaches players what language rhythm sounds like independent of topic.

Pipeline Changes

  • Add a new ingestion script, probably scripts/build-sentence-manifest.mjs, that downloads source metadata and writes a normalized manifest.
  • Store audio on same-origin static hosting or R2 when the source license allows redistribution; otherwise keep remote URLs only if the source supports stable media playback.
  • Filter clips by duration >= 5 && duration <= 10, valid transcript, supported language, and playable media headers.
  • Add a “no spoiler” filter for transcripts containing country names, city names, language names, or obvious demonyms.
  • Normalize metadata fields: url, source_url, source, license, attribution, dur, text, lang, country, region, family, geo.
  • Generate an audit report with rejected counts per language so the app does not silently ship weak or empty languages.

Practical Next Step

Build a proof-of-concept with 8-10 Common Voice languages already present in LinguaGuessr: English, French, German, Spanish, Italian, Portuguese, Dutch, Swedish, Russian, and Arabic. Pull 30 validated clips per language, filter to 5-10 seconds, and compare game feel against the existing Lingua Libre manifest. If that works, expand language coverage and add source-specific attribution UI only where the chosen license requires it.
